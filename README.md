# ГИС-платформа «Сенсорный портрет города»

Прототип интерактивной карты для сбора субъективных городских данных:
- эмоциональные якоря;
- памятные метки;
- личные навигаторы;
- лучшее место;
- фото-якоря;
- инфраструктурный отклик;
- скрытые сокровища.

## Что реализовано

- Подложка карты: `OpenStreetMap` (через `Leaflet`).
- Переключаемые тематические слои (7 слоев).
- Предустановленные шаблоны карт + сохранение пользовательских шаблонов.
- Добавление точек и маршрутов с параметрами по каждому слою.
- Привязка геометрии:
  - вручную (клики на карте, перетаскивание маркера);
  - автоматическая (кнопка геолокации браузера).
- Демо-авторизация (e-mail + пароль) с локальным хранением.
- Отзывы и оценки к объектам.
- Сохранение чужих меток/маршрутов в профиль.
- Все данные хранятся локально в `localStorage`.

## Быстрый запуск

Откройте файл `index.html` в браузере.

Для корректной работы загрузки фото и API браузера рекомендуется запуск через локальный сервер:

```bash
cd /Users/bataev.aa/Documents/New\ project
python3 -m http.server 8080
```

Далее откройте `http://localhost:8080`.

## Ограничения текущего прототипа

- Авторизация и данные хранятся локально (без серверной БД и шифрования пароля).
- OAuth через соцсети пока не подключен.
- Модерация контента на уровне ролей администратора пока не реализована.

## Что нужно для production

- Backend API + БД (PostgreSQL/PostGIS).
- Нормальная аутентификация (JWT/OAuth2) и хэширование паролей.
- Политики модерации и роли (user/moderator/admin).
- Хранение фото в object storage (S3-совместимое).
- Логи, аудит, rate limit, валидация и защита от злоупотреблений.
